二、写作命令、布局 (Layout:Page/Post) 、模板 (Scaffold) 介绍

写作命令(指令)

Hexo 中按作用划分有两种类型的命令(指令)：系统命令、写作命令 。

命令均是在 [BlogRoot]/ 根目录下执行，推荐使用 VSCode 作为集成开发环境进行博客开发。

【写作命令】、【系统命令】介绍

系统命令针对的主要是项目的创建、编译生成、运行、清理、部署等步骤，以下介绍最常用的三个系统命令。

总结：hexo三件套一步执行命令

  1. hexo clean
    可简写为：

    清除缓存文件 (db.json) 和已生成的静态文件 (public)。
    在某些情况（更换主题后、更改配置文件后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

  2. hexo generate
    可简写为：

    生成静态文件(public)。
    public:
    最终项目编译后文件存储的地方，发布(运行)后直接访问的静态资源内容。
    例如，当你在[BlogRoot]/source/_posts/my.md 创建了一篇markdown文章,执行hexo g命令后，系统将其编译为一篇xxx.html的网页并放置在[BlogRoot]/public/posts/xxx.html,之后当你执行hexo s, 你就在域名后跟上public下的资源定位路径，如：http://localhost:4000/ post/xxx.html 即可。

  3. hexo server
    可简写为：

    启动服务器。默认情况下，访问网址为：http://localhost:4000/。

Hexo 写作命令 (命令就一个，但搭配多种参数，有多种效果)，用于生成新文章(post)、新页面(page)。在此主要介绍页面、文章的概念及利用命令生成它们。

命令行参数语法格式说明

命令行参数语法格式说明，各系统之间也有差别。一般采用格式如下：

  • 尖括号< >：: 必选内容，实际使用去除< >，并将里面的内容按要求替换(或不替换)为需要的内容
  • 方括号[ ]： 可选内容，实际使用时去除[ ],可选内容意味着可以省略，不写该内容。
  • 大括号{ }： 必选内容，内部使用(被其它符号包含)，实际使用时去除{ }包含此处允许使用的参数
  • 小括号( )： 指明参数的默认值，只用于{ }中， 实际使用时去除( )
  • 竖线 | ： 可用于 < >,{ },[ ] 中，用于分隔多个互斥参数，含义为“或”，使用时只能选择一个
  • 省略号…： 任意多个参数

布局（Layout）

Hexo 有三种默认布局：post、page 和 draft。在创建这三种不同类型的文件时，它们将会被保存到不同的路径；而您自定义的其他布局和 post 相同，都将储存到 source/_posts 文件夹，即都显示为博客文章。
注意：post 、 page 、 draft 以及你的自定义布局，实际就是 [blog根目录]/scafflods 的模板。当执行 hexo new page "page_name" 时，是调用的 scaffolds/page.md 为模板生成的 [blog根目录]/source/page_name 页面

执行对应布局的写作命令 hexo new ... 后，文章或页面存储的位置如下：

布局 存储路径(hexo new) 编译后路径(hexo g) 解释
post source/_posts public/posts post 为博客文章，即本文效果
page source public page 为页面，首页/标签页/分类页等都是页面布局
draft source/_drafts public/drafts 草稿，实际使用价值不多，因为可以用隐藏的文章代替

执行hexo s后，访问各种文章或页面：

布局 文章或页面实例 网页访问URL
post 一篇 title.md 文章编译到 public/posts/title.html http://localhost:4000/ posts/title.html
page source/tags/ 页面编译到 public/tags/ http://localhost:4000/ tags

三、标签页、分类页以及文章生成

上面讲了模板、布局和生成命令后，就应该根据实际需求生成我们的页面和文章了，主要内容如下：

  • 页面：页面前页参数（Page Front-matter）、分类页（categories）、标签页（tags）
  • 文章：文章前页参数（Post Front-matter）

Front-matter

Front-matter 是 markdown 文件最上方以 --- 分隔的区域，用于指定不同文章或页面的效果。我们根据页面或文章将其分为两种：

  • Page Front-matter 用于 页面 配置
  • Post Front-matter 用于 文章 配置

  2. 将下载的 hexo-butterfly-tag-plugins-plus.zip 解压得到资源文件夹,下文以 [tag_plugins_file] 指代该文件夹。

  3. 将 [tag_plugins_file]\lib\scripts 目录下的文件放到 [Blogroot]\themes\butterfly\scripts\tag\ 文件夹内。

  4. 将 [tag_plugins_file]\lib\style 目录下的文件放到 [Blogroot]\themes\butterfly\source\css\tags\ 文件夹内。

  5. 修改 [Blogroot]\_config.butterfly.yml 的 inject 配置项，添加 CDN 依赖项。由于 issues 写入 timeline 和 site-card 标签要用到 jquery ，请务必根据注释指示的版本决定是否添加。

  6. 考虑到hexo自带的markdown渲染插件 hexo-renderer-marked 与外挂标签语法的兼容性较差，建议您将其替换成 hexo-renderer-kramed

  7. 文章中外挂标签的使用请参阅下文。
