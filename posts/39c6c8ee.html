Visual Studio 2019 + QT 5.14.2 + QGIS 3.16 （RELEASE版）

发表于2022-11-25|更新于2022-12-01|QGIS开发
|字数总计:4.4k|阅读时长:15分钟|阅读量:|评论数:

本文章记录近日进行的 QGIS 二次开发 的环境配置历程。一方面就我所用的环境与版本，给出一个能够正常调用与运行的配置；另一方面就为什么这样配置（原理），以及如何处理配置过程中出现的 Bug 进行解释。相信你如果理解这里面的原理，就算你配置过程出错了，你也可以自己去独立解决了。

本教程版本需知：2022 年 11 月有效。版本说明如下：

  • 本机系统：Window 10 系统
  • Visual Studio 版本：Visual Studio 2019 Community 版
  • QT 版本：QT 5.14.2 版本
  • QGIS版本： 3.16 LTR 版本，具体至 阶段版本号 为 3.16.16。

之间的版本关系具体看下面各个软件的安装说明。

一、Visual Studio 2019 集成开发环境安装

Visual Studio 2019 安装

Visual Studio 可以作为 C++ 的集成开发环境，但不是唯一的。比如 QT 官方专门针对 QT 程序开发提供了 QT Creator 这个集成开发环境，所以我们也可以利用 QT Creator 进行 QGIS 的二次开发（当然还有很多其它的集成开发环境，如 VSCode 等也能进行 C++ 开发）。之所以选择 Visual Studio 主要原因是这个软件发展的足够好， 能够方便的引入 QT 开发库和 QGIS 开发库。另外，Visual Studio 的版本只要是可以正常使用的版本就行，2015，2017，2019 均可，不过目前 2019 已经很稳定了，推荐使用。

总之，集成开发环境（IDE）只是一个工具，用来运行一种语言的代码的环境。对 Visual Studio 而言，它本身没有 QT 开发的库 和 QGIS 开发的库，本身是不可以进行 QGIS 二次开发的，但我们可以指定这个工具去将这些库引进来，那么你在写 C++ 的代码的时候，就可以去调 QGIS 的功能和方法了。


 正如上面图所示，我们的目标其实非常简单：把需要用到的 QT 和 QGIS 的库引进 Visual Studio。(其它 IDE 同理，所以用 QT Creator 和 VsCode 也是可以进行 QGIS 二次开发的，只要引入开发需要的库就行)
 这样，成功引进来并可以在代码中像调标准 C/C++ 库那样正确不报错的使用就是我们的开发环境配置成功了。

二、QT 5.14.2 安装

QT 5.14.2 的安装

Visual Studio 的版本与 QT 插件（用于引入 QT 库）有一定关系，VS 2019 推荐使用 QT 5.14 版本以上。QT 与 QGIS 之间有直接的版本要求,主要原因是：QGIS 是基于 QT 开发的。
本文章选用的是 QGIS 3.16 的 LTR（长期发布）版本，其要求的是 QT 5 以上的版本（QGIS 3.16 进行窗体开发调用的是 QGIS 5 提供的库）。
那么如何获取 QT 5 以上的开发环境（简而言之，获取 QT 5 以上的库），其实有两种方法：

  1. QT 官网下载 QT 5，这里推荐使用 QT 5.14.2
  2. 不用下载，直接利用 OSGeo4W 下载的 QGIS 3.16 自带的 QT 环境

所以说，QT 5.14.2 是可以不用下载的，直接下载 QGIS 3.16 就行。
本文的配置是在 QGIS 二次开发中引用自己下载的 QT 环境（因为当时候不知道 QGIS 自带 QT 5 环境 QaQ），即第1种方法（第2种方法不用下载直接引用，也是可以的）。

  1. 下载 QT 5.14.2

1.Release 和 Debug 的设置
  • Release：发布版。特指源码以 Release 方式进行编译，不具备调试功能，但可以快速运行。
  • Debug：调试版。源码以 Debug 方法进行编译，具备调试功能，所以编译生成内容相比 Release 更多，运行速度较其慢一些。

    Release 版和 Debug 版生成的 lib 文件 和 dll 文件是有区别的，不能混合调用。
    此次下载的 QT 和 QGIS 都是 Release 版本（即发布版），所以我们要以 Release 方式编译我们二次开发的项目。
